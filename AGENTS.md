# AI408 Frontend

## 项目概述
AI408 前端是一个面向 408 考研刷题场景的 Vue 应用，负责登录、题库导入入口、随机练习、背题、错题本、模拟考试、AI 讲解展示，以及与后端真实 API 的联调。

本仓库是前端仓库，对应后端仓库位于：

- `D:\02code\02IdeaProjects\AI408`

涉及接口契约、练习会话、导题、AI 流、图片题等功能时，需要同时参考后端仓库。

## 技术栈
- Vue 3
- Vue Router 4
- Vite 8
- Tailwind CSS 3
- 原生 `fetch` API 封装
- `localStorage` 保存 token
- 轻量状态管理放在 `src/stores`

## 项目结构
```text
src/
├── api/           # 后端接口封装
├── components/    # 通用组件
├── data/          # UI 配置和少量静态展示数据
├── pages/         # 页面级组件
├── stores/        # 轻量状态管理
├── styles/        # 样式
└── router.js      # 路由配置
```

## 主要页面
- `/login` 登录页
- `/` 首页
- `/practice` 刷题 / 背题
- `/review` 练习复盘
- `/mistakes` 错题本
- `/favorites` 收藏题目
- `/exam` 模拟考试
- `/exam/records/:id` 考试记录回顾
- `/admin/import` 管理员题库导入

## 前端业务约定
- 前端只认后端统一返回：`ApiResponse<T> { code, message, data }`
- 成功判断以 `code === '200'` 为准
- token 存在 `localStorage`
- access 失效时优先 refresh，再失败才回登录页
- 文件下载链接不能直接使用后端返回的相对路径，必须补全为后端绝对地址
- 练习、复盘、考试错题重做等跨页动作，优先走 `src/stores` 中的共享状态

## 编码规范
- 页面逻辑放 `pages/`，接口调用放 `api/`
- 组件命名使用 PascalCase
- 工具函数与局部帮助函数使用 camelCase
- 新增业务状态时，优先复用已有 `stores/auth.js` 与 `stores/study.js`
- 不要把真实业务重新塞回 `src/data/ai408.js` 的 mock 结构中
- 中文文案要保持可读，避免乱码
- 保留现有 UI 风格，不要为了重构而重写整个页面

## 已知实现特点
- 刷题和背题必须使用同一批题目，区别只在展示方式
- 背题模式要直接高亮正确答案，并显示解析
- 错题本不是单独题目详情页，而是“进入错题练习”
- 模拟考试与练习页风格接近，但数据流独立
- AI 讲解通过流式渲染增量文本
- 图片题如果有 `stemImageUrl`，页面需要支持预览和点击查看原图

## 注意事项
- 前端资源下载地址要通过 `VITE_API_BASE_URL` 补全
- 改接口字段前要确认后端 DTO 是否已同步
- 改练习页时要同时留意：
  - 刷题模式
  - 背题模式
  - 练习结束
  - 复盘跳转
  - 错题重做
- 改考试页时要同时留意：
  - 倒计时
  - 收藏
  - 交卷
  - 弃考
  - 考试记录回顾
- 不要擅自恢复用户已调整过的样式或交互

## 环境变量
- `VITE_API_BASE_URL`

默认本地通常指向：

- `http://localhost:8080`

## 本地开发建议
- 先看页面对应的 `src/api` 和 `src/stores`
- 如果页面异常，先确认是：
  - 接口数据不对
  - 路由状态不对
  - 资源 URL 拼接错误
  - 文案或条件渲染问题
- 提交前至少跑一次：
  - `npm run build`

## 给 Codex 的建议
- 先确认当前改动是纯前端，还是要联动后端仓库
- 优先复用现有页面和状态流，不要平地再起一套
- 涉及“错题重做 / 复盘 / 考试回顾”的功能，尽量走统一的待创建练习配置
- 如果发现问题根因在后端返回结构、数据库状态或文件地址，不要硬在前端绕过去，明确指出需要联动后端
